# 1. ОБЩЕЕ ОПИСАНИЕ СИСТЕМЫ

## 1.1. Наименование и назначение системы

Разрабатываемая система представляет собой цифровую платформу для автолюбителей «АвтоПомощник» (далее — Платформа), предназначенную для комплексного учёта и управления информацией, связанной с владением и эксплуатацией автомобилей.

Платформа реализована в виде двух клиентских приложений:
- веб-приложение для работы через браузер;
- мобильное приложение для устройств на базе операционных систем Android и iOS.

Оба клиентских приложения взаимодействуют с единой серверной частью посредством REST API, что обеспечивает синхронизацию данных между устройствами пользователя.

## 1.2. Функциональные требования

### 1.2.1. Модуль авторизации

Система должна обеспечивать:
- регистрацию новых пользователей с указанием адреса электронной почты и пароля;
- аутентификацию пользователей по адресу электронной почты и паролю;
- восстановление пароля через электронную почту;
- защиту пользовательских сессий с использованием JWT-токенов.

### 1.2.2. Модуль «Гараж»

Модуль предназначен для управления списком автомобилей пользователя и должен обеспечивать:
- добавление нового автомобиля с указанием марки, модели, года выпуска, VIN-номера, государственного регистрационного знака;
- редактирование информации об автомобиле;
- удаление автомобиля из списка;
- просмотр списка всех автомобилей пользователя;
- загрузку фотографии автомобиля.

### 1.2.3. Модуль «Сервисная книжка»

Модуль предназначен для ведения истории технического обслуживания и ремонтов и должен обеспечивать:
- добавление записи о проведённых работах с указанием даты, пробега, типа работ, описания и стоимости;
- классификацию работ по типам: плановое ТО, ремонт, замена расходников, шиномонтаж, прочее;
- привязку записей к конкретному автомобилю;
- просмотр истории обслуживания с возможностью фильтрации и сортировки;
- редактирование и удаление записей.

### 1.2.4. Модуль «Расходы»

Модуль предназначен для учёта финансовых затрат на автомобиль и должен обеспечивать:
- добавление записей о заправках с указанием даты, объёма топлива, стоимости, пробега;
- добавление записей о прочих расходах: страховка, налоги, штрафы, парковка, мойка;
- автоматический расчёт среднего расхода топлива;
- формирование статистики расходов по категориям и периодам;
- визуализацию данных в виде графиков и диаграмм.

### 1.2.5. Модуль «Напоминания»

Модуль предназначен для уведомления пользователя о предстоящих событиях и должен обеспечивать:
- создание напоминаний по дате (страховка, техосмотр, замена документов);
- создание напоминаний по пробегу (замена масла, фильтров, ремня ГРМ);
- отправку уведомлений в приложении;
- отображение списка активных напоминаний.

### 1.2.6. Модуль «Справочник»

Модуль содержит справочную информацию и должен обеспечивать:
- каталог марок и моделей автомобилей;
- автозаполнение при добавлении автомобиля.

## 1.3. Нефункциональные требования

### 1.3.1. Требования к производительности
- время отклика сервера при стандартных операциях не должно превышать 500 мс;
- система должна поддерживать одновременную работу не менее 100 пользователей.

### 1.3.2. Требования к безопасности
- хранение паролей в зашифрованном виде с использованием алгоритма bcrypt;
- передача данных по защищённому протоколу HTTPS;
- валидация всех входящих данных на стороне сервера.

### 1.3.3. Требования к совместимости
- веб-приложение должно корректно работать в браузерах Chrome, Firefox, Safari, Edge последних двух версий;
- мобильное приложение должно поддерживать Android 8.0+ и iOS 13.0+.

### 1.3.4. Требования к масштабируемости
- архитектура системы должна допускать горизонтальное масштабирование серверной части.

## 1.4. Пользователи системы

Целевая аудитория платформы — владельцы личных автомобилей, заинтересованные в систематизации информации о своих транспортных средствах.

В системе предусмотрена единственная роль пользователя — **автовладелец**. Административный функционал в рамках данного проекта не реализуется, поскольку система ориентирована на индивидуальное использование.

## 1.5. Границы системы

В рамках дипломного проекта НЕ реализуются:
- интеграция с внешними сервисами (ГИБДД, страховые компании);
- социальные функции (обмен данными между пользователями);
- маркетплейс запчастей или услуг;
- push-уведомления на мобильных устройствах (только in-app уведомления);
- мультиязычность (только русский язык).

Данные ограничения обусловлены объёмом дипломной работы и могут быть сняты при дальнейшем развитии проекта.
